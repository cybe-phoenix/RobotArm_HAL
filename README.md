# 基于 STM32 的六自由度机械臂示教与回放控制系统 V1.2

> 基于 **STM32F103C8T6 + HAL 库** 实现的六自由度机械臂控制系统。系统支持本地电位器示教、动作点记录与平滑回放、OLED 状态显示、USART 串口协议控制，以及 PyQt 图形化上位机控制与串口调试辅助功能。

## 1. 项目简介

本项目面向六自由度舵机机械臂的基础控制需求，采用 STM32F103C8T6 作为主控芯片。五个关节由电位器或上位机 PWM 指令控制，夹爪作为第六执行机构独立控制。

系统具有两种控制入口：

- **本地示教控制**：通过五个电位器实时调整五个关节姿态，并使用四个按键完成记录、回放、夹爪开合和清空记录。
- **PC 上位机控制**：通过 USB-TTL 与 USART1 通信，使用 PyQt 图形化界面完成模式切换、五关节滑条控制、夹爪控制、回中位、记录、回放与清空记录。

动作记录统一保存当前关节的 **PWM 目标值** 与夹爪状态，因此无论动作来自电位器示教还是 PyQt 上位机，都可以被记录并回放。

## 2. 当前功能

### 2.1 STM32 本地控制功能

- 五个电位器实时控制五个关节舵机。
- KEY1：清空已记录动作点。
- KEY2：回放已记录动作序列。
- KEY3：控制夹爪打开或闭合。
- KEY4：记录当前动作点。
- 相邻动作点之间采用线性插值，使回放过程更平滑。
- 动作点同步保存夹爪状态，回放时自动恢复夹爪开合状态。
- 对关节 PWM 进行限幅，减少机械结构到达极限位置的风险。

### 2.2 OLED 显示功能

0.96 英寸 I2C OLED 可显示：

- 当前控制模式：`Manual`、`PC`、`Playback`。
- 当前已记录动作点数量。
- 夹爪状态：`Open` 或 `Close`。
- 五个关节当前 PWM 值，例如：`[150,150,150,150,150]`。

### 2.3 串口协议功能

- 支持 PC / Manual 控制模式切换。
- 支持五关节 PWM 目标值下发。
- 支持夹爪打开、闭合和五关节回中位。
- 支持通过串口执行记录、回放和清空记录。
- 支持对非法命令或非 PC 模式下的关节控制命令返回错误提示。

### 2.4 PyQt 上位机功能

- 串口扫描、连接和断开。
- PC 控制模式与手动模式切换。
- 五个关节 PWM 滑条控制。
- 夹爪打开、闭合控制。
- 五关节回中位。
- 记录当前点、回放动作、清空记录。
- 串口助手风格的配置与调试功能：
  - 数据位、校验位、停止位、流控配置；
  - 接收 ASCII / Hex 显示；
  - 自动换行、显示发送、显示时间；
  - 发送 ASCII / Hex；
  - 自动重发与重发间隔；
  - `Line by Line` 逐行发送；
  - 手动发送输入框。

## 4. 硬件连接与引脚分配

### 4.1 关节与夹爪 PWM 输出

| 功能 | STM32 引脚 | 定时器通道 | 说明 |
|---|---:|---|---|
| 关节 1 | PA6 | TIM3_CH1 | 第 1 路关节舵机 |
| 关节 2 | PA7 | TIM3_CH2 | 第 2 路关节舵机 |
| 关节 3 | PB0 | TIM3_CH3 | 第 3 路关节舵机 |
| 关节 4 | PB1 | TIM3_CH4 | 第 4 路关节舵机 |
| 关节 5 | PB6 | TIM4_CH1 | 第 5 路关节舵机 |
| 夹爪 | PB7 | TIM4_CH2 | 第 6 执行机构 / 夹爪舵机 |

### 4.2 电位器 ADC 输入

| 功能 | STM32 引脚 | ADC 通道 | 控制对象 |
|---|---:|---|---|
| 电位器 1 | PA0 | ADC1_IN0 | 关节 1 |
| 电位器 2 | PA1 | ADC1_IN1 | 关节 2 |
| 电位器 3 | PA2 | ADC1_IN2 | 关节 3 |
| 电位器 4 | PA3 | ADC1_IN3 | 关节 4 |
| 电位器 5 | PA4 | ADC1_IN4 | 关节 5 |

### 4.3 独立按键输入

按键采用 **低电平有效** 接法：GPIO 与按键一端相连，按键另一端接 GND，GPIO 配置为上拉输入。

| 按键 | STM32 引脚 | 功能 |
|---|---:|---|
| KEY1 | PB12 | 清空已记录动作点 |
| KEY2 | PB13 | 回放动作序列 |
| KEY3 | PB14 | 切换夹爪开合 |
| KEY4 | PB15 | 记录当前动作点 |

### 4.4 OLED 连接

OLED 使用 I2C2，避免占用 PB6 / PB7 的舵机 PWM 引脚。

| OLED 引脚 | STM32 引脚 | 说明 |
|---|---:|---|
| VCC | 3.3 V | OLED 电源 |
| GND | GND | 共地 |
| SCL | PB10 | I2C2_SCL |
| SDA | PB11 | I2C2_SDA |

### 4.5 USART1 与 USB-TTL 连接

| USB-TTL | STM32 引脚 | 说明 |
|---|---:|---|
| TXD | PA10 | USB-TTL 发送接 STM32 USART1_RX |
| RXD | PA9 | USB-TTL 接收接 STM32 USART1_TX |
| GND | GND | 必须共地 |

> 串口线必须交叉连接：`USB-TTL TXD → PA10`，`USB-TTL RXD → PA9`。

## 5. 系统工作模式

### 5.1 Manual 模式：本地示教

- 系统默认进入 `Manual` 模式。
- 主循环读取五路 ADC 数据，实时更新五个关节舵机。
- 可使用 KEY1~KEY4 完成清空、回放、夹爪开合、记录等操作。
- 即使串口已经连接，只要未进入 PC 模式，PC 发送 `#J,...!` 不会接管五关节控制。

### 5.2 PC 模式：上位机控制

- 通过 `#MODE,PC!` 进入 PC 控制模式。
- 电位器不再更新五关节 PWM，避免与上位机命令抢占控制权。
- PyQt 滑条下发五关节 PWM 值。
- 可通过上位机或串口命令控制夹爪、回中位、记录、回放和清空记录。

### 5.3 Playback 状态：动作回放

- 由 KEY2 或 `#PLAY!` 触发。
- 机械臂依次经过每一对相邻动作点，并采用线性插值平滑过渡。
- 回放时 OLED 显示 `Playback`。
- 当前版本回放函数为阻塞式执行；回放期间不建议持续下发新的运动命令。

## 7. 核心模块说明

### 7.1 `robot_servo`

负责五个关节的 PWM 输出与状态维护。

主要功能：

- 启动 TIM3 / TIM4 对应 PWM 通道。
- 五关节回中位。
- 将 ADC 数值映射为关节 PWM。
- PWM 限幅。
- ADC 控制时实时更新关节 PWM。
- 串口控制、回放控制时设置关节 PWM。
- 提供当前五关节 PWM 读取接口，供 OLED 与记录模块使用。

### 7.2 `robot_gripper`

负责夹爪舵机的开合控制。

主要功能：

- 控制夹爪打开。
- 控制夹爪闭合。
- 对夹爪 PWM 进行安全范围限制。
- 按当前设计完成动作后停止夹爪 PWM，降低持续受力和发热风险。

### 7.3 `robot_key`

负责四个按键的扫描与消抖。

| 按键 | 调用功能 |
|---|---|
| KEY1 | `Record_Clear()` |
| KEY2 | `Playback_Record()` |
| KEY3 | `Gripper_Open()` / `Gripper_Close()` |
| KEY4 | `Record_Current_Point()` |

### 7.4 `robot_record`

负责动作点记录与回放。

每个动作点保存：

```text
五个关节当前 PWM 值 + 当前夹爪状态
```

主要功能：

- 保存当前五关节 PWM 值。
- 保存当前夹爪状态。
- 清空已记录动作点。
- 判断动作点数量是否满足回放条件。
- 对相邻动作点进行 PWM 线性插值。
- 在回放节点切换对应夹爪状态。

### 7.5 `robot_oled`

负责 SSD1306 OLED 的显示驱动与状态页刷新。

显示信息：

```text
Mode: Manual / PC / Playback
Points: 003
Grip: Open / Close
[150,150,150,150,150]
```

### 7.6 `robot_uart`

负责 USART1 命令帧接收、解析和响应。

主要功能：

- 基于接收中断逐字节接收串口数据。
- 使用 `#` 作为帧起始，`!` 作为帧结束。
- 解析控制模式、关节 PWM、夹爪、回中位、记录、回放、清空记录等命令。
- 通过请求标志将记录、回放、清空等主流程任务交给 `main.c` 统一调度。

### 7.7 `main.c`

负责系统初始化与模块调度。

主要流程：

1. 初始化 HAL、时钟、GPIO、DMA、ADC、TIM、I2C2、USART1。
2. 初始化舵机并启动 PWM。
3. 启动 ADC + DMA 连续采样。
4. 初始化 OLED 与 USART1。
5. 主循环执行串口任务、按键任务、记录/回放/清空请求处理。
6. 在 `Manual` 模式且非回放状态下，根据 ADC 更新关节 PWM。
7. 周期刷新 OLED 状态信息。

## 8. STM32 外设配置要点

### 8.1 PWM 定时器

| 外设 | 参数 |
|---|---|
| TIM3 | CH1~CH4：PA6、PA7、PB0、PB1 |
| TIM4 | CH1~CH2：PB6、PB7 |
| Prescaler | 719 |
| Period | 1999 |
| PWM 周期 | 20 ms，约 50 Hz |
| 计数分辨率 | 1 count = 10 μs |
| 典型中位值 | CCR = 150，对应约 1.5 ms |

### 8.2 ADC + DMA

| 参数 | 配置 |
|---|---|
| ADC | ADC1 |
| 通道 | CH0~CH4 |
| 扫描模式 | Enable |
| 连续转换 | Enable |
| 转换数量 | 5 |
| DMA | DMA1_Channel1 |
| DMA 模式 | Circular |
| 数据宽度 | Halfword |

初始化时需保证：

```c
MX_GPIO_Init();
MX_DMA_Init();
MX_ADC1_Init();
```

其中 `MX_DMA_Init()` 应在 `MX_ADC1_Init()` 之前。

### 8.3 I2C2

| 项目 | 配置 |
|---|---|
| I2C 外设 | I2C2 |
| SCL | PB10 |
| SDA | PB11 |
| OLED | SSD1306，128×64 |

### 8.4 USART1

| 项目 | 配置 |
|---|---|
| TX | PA9 |
| RX | PA10 |
| 波特率 | 115200 |
| 数据位 | 8 |
| 校验 | None |
| 停止位 | 1 |
| 流控 | None |
| 中断 | USART1 Global Interrupt Enable |

## 9. 使用说明

### 9.1 上电与本地示教

1. 确认舵机电源、STM32 电源和 GND 共地。
2. 上电后，五个关节进入中位附近。
3. OLED 显示当前模式、记录点数、夹爪状态和五关节 PWM。
4. 旋转五个电位器，调整机械臂姿态。
5. 按 KEY4 记录当前姿态。
6. 调整到新的姿态后再次按 KEY4，可记录多个动作点。
7. 按 KEY3 切换夹爪状态；夹爪状态会与动作点一同保存。
8. 按 KEY2 回放动作序列。
9. 按 KEY1 清空所有记录点。

### 9.2 PyQt 上位机控制

1. 将 USB-TTL 与 STM32 USART1 按交叉方式连接。
2. 关闭可能占用串口的串口助手或其他上位机程序。
3. 启动 PyQt 上位机主程序。
4. 刷新串口，选择正确的 COM 端口。
5. 串口参数设置为：

```text
115200 / 8 / None / 1 / None
```

6. 点击“连接”。
7. 按 STM32 的复位键后，可在日志区观察到：

```text
Robot UART Ready
```

8. 点击“进入 PC 控制模式”。
9. 使用 J1~J5 滑条控制五个关节。
10. 使用夹爪、回中位、记录、回放、清空按钮执行对应操作。
11. 点击“回到手动模式”后，电位器重新获得五关节控制权。

## 10. 串口通信协议

### 10.1 帧格式

```text
#命令,参数1,参数2,...!
```

- 帧起始字符：`#`
- 帧结束字符：`!`
- 推荐以 ASCII 文本发送。
- 上位机发送时可附加 `\r\n`，STM32 仅以 `#` 和 `!` 作为命令帧边界。

### 10.2 控制命令

| 命令 | 示例 | 说明 |
|---|---|---|
| 进入 PC 模式 | `#MODE,PC!` | 上位机接管五关节控制 |
| 回到手动模式 | `#MODE,MANUAL!` | 电位器重新控制五关节 |
| 五关节控制 | `#J,150,150,150,150,150!` | 设置 J1~J5 PWM 值 |
| 夹爪打开 | `#G,0!` | 打开夹爪 |
| 夹爪闭合 | `#G,1!` | 闭合夹爪 |
| 关节回中位 | `#MID!` | 五个关节回到 PWM 150 |
| 记录当前点 | `#REC!` | 保存当前 PWM 与夹爪状态 |
| 回放动作 | `#PLAY!` | 回放已记录动作点 |
| 清空记录 | `#CLR!` | 清空动作点 |

### 10.3 典型响应

| 操作 | 典型响应 |
|---|---|
| 进入 PC 模式 | `OK:MODE PC` |
| 回到手动模式 | `OK:MODE MANUAL` |
| 夹爪打开 | `OK:GRIP OPEN` |
| 夹爪闭合 | `OK:GRIP CLOSE` |
| 回中位 | `OK:MID` |
| 记录当前点 | `OK:REC` |
| 回放动作 | `OK:PLAY` |
| 清空记录 | `OK:CLR` |
| 手动模式下发送 `#J` | `ERR:NOT PC MODE` |
| 未识别命令 | `ERR:UNKNOWN CMD` |

### 10.4 `#J` 应答说明

为避免拖动滑条时大量 `OK:J` 返回占用串口带宽，工程中可通过：

```c
#define UART_ACK_J_ENABLE  0
```

关闭五关节控制命令的逐条应答。该设置下，`#J,...!` 正常执行但不返回 `OK:J`。

## 12. 关键参数

### 12.1 五关节 PWM 范围

当前五关节使用的统一限幅范围：

```c
#define SERVO_MIN 75
#define SERVO_MAX 225
#define SERVO_PWM_DEADBAND 2
```

说明：

- `75~225` 对应约 `0.75~2.25 ms` 的 PWM 高电平范围。
- 具体机械结构存在差异，部分关节可能需要使用更窄的安全范围。
- 若机械臂碰撞机械限位，应优先缩小对应关节的范围，而不是继续扩大统一范围。

### 12.2 主循环周期

当前主循环保留：

```c
HAL_Delay(10);
```

该设置在控制平滑性、按键扫描和 OLED 刷新之间取得了当前硬件下的平衡。

## 14. 注意事项

1. **舵机电源不要直接由 STM32 开发板 3.3 V 供电。** 建议使用独立、足够电流能力的 5 V 电源。
2. STM32、舵机电源和 USB-TTL 必须共地，否则 PWM 或串口通信可能异常。
3. 第一次上电或调整 PWM 范围前，应使机械臂处于安全姿态，避免因初始位置偏差撞击机械限位。
4. 使用 USB-TTL 时应确保其逻辑电平与 STM32 兼容，推荐使用 3.3 V TTL 电平。
5. 串口连接失败时，先关闭串口助手、PyQt 或其他占用同一 COM 口的软件。
6. 机械臂运动时不要强行掰动正在受力的舵机，避免齿轮损坏。
7. 当前回放为阻塞式实现，回放过程中不要重复触发回放或持续发送新的执行命令。
